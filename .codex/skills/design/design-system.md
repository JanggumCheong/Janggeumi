# 장금이 디자인 시스템

> 이 문서는 `design` 스킬의 기준이다. 목업·컴포넌트를 만들 때 여기 정의된 토큰·규칙을 따른다.
> 임의로 색·간격·폰트를 지어내지 않는다. 일관성이 최우선.

---

## 0. 디자인 철학 — "식자재 고민 시간을 줄인다"

**핵심 비유:** 주식은 원래 딱딱하고 어렵게 느껴졌지만, 토스는 그걸 **유려하고 심플하며 직관적이고 현대적으로** 풀어내 성공했다. 장금이도 똑같다 — 식재료 관리는 원래 번거롭고 촌스럽게 느껴지지만, 우리는 그걸 **유려·심플·직관·현대**로 풀어낸다. "장금이"는 친근함을 빌려오는 매개체일 뿐, 디자인까지 옛날 그대로 표현하는게 아니라 시각 언어는 토스처럼 세련되게 간다.

토스(토스증권)를 **고민 지점이 생길 때만** 참고한다. 토스가 후발주자로 성공한 이유는 유려하고 **누구나 쉽고 간편하게 쓰는 직관적 디자인**이었다. 장금이도 같은 목표다: 살림 초보가 헤매지 않고 바로 확신을 얻는다.

**토스에서 가져오는 원칙 (판단이 애매할 때 이 기준으로 결정):**

1. **한 화면 한 할 일.** 한 화면에서 사용자가 할 결정은 하나. 선택지를 늘어놓지 말고 다음 행동을 명확히.
2. **여백이 정보다.** 빽빽하게 채우지 않는다. 넉넉한 여백으로 시선을 유도.
3. **큰 타이포 + 명확한 위계.** 제목은 크고 굵게, 보조는 회색으로 확실히 구분.
4. **색은 절제, 포인트는 하나.** 화면 대부분 무채색 + 흰 배경. 강조색은 액션·핵심 값에만.
5. **큰 라운드, 부드러운 카드.** 날카롭지 않은 둥근 모서리, 옅은 그림자.
6. **버튼은 크고 확실하게.** 주요 액션은 폭 넓은 채움 버튼. 탭 영역 충분히.
7. **숫자·상태를 즉시 읽히게.** 별점·평가수·"베스트" 같은 핵심 신호는 한눈에.

**장금이 고유색 (토스와 다른 점):** 토스는 파랑 핀테크 톤이지만, 장금이는 **세이지 그린**을 포인트로 쓴다. **배경은 흰색/거의 흰색(`#FBFBFA`)** 이다 — 시안이 전부 흰 배경이므로 아이보리·베이지 배경을 깔지 않는다. 장금이의 "따뜻함"은 배경색이 아니라 **일러스트(한복 캐릭터)·카피·그린 포인트**로 낸다. 레이아웃·여백·위계·간결함의 원리는 토스와 동일하게 간다.

**⚠️ 절대 원칙 — 현대적으로. 장금이는 매개체일 뿐이다.**

이 서비스의 타겟은 **지금 시대에 살림을 잘하고 싶은 사람**이다. "장금이"는 친근한 살림 고수라는 정서를 빌려오는 **매개체**일 뿐, 디자인의 목적지가 아니다. 다음을 지킨다:

- **전통/사극/한복 느낌으로 흐르지 않는다.** 한지 텍스처, 붓글씨체, 자개·단청·전통문양, 세피아·황토 과채도, 두루마리 장식 등 "옛날" 시각 요소 **금지**.
- 톤은 어디까지나 **모던·미니멀·깔끔**하다. 토스처럼 2020년대 모바일 앱의 감각(넉넉한 여백, 산세리프, 큰 라운드, 절제된 색)을 유지한다.
- 세이지 그린·아이보리는 "전통색"이 아니라 **차분한 현대 뉴트럴**로 쓴다. 저채도로 유지해 세련되게. 채도를 높여 "촌스러운 자연색"이 되지 않게 한다.
- 장금이 캐릭터/카피는 **가볍고 위트 있게** 등장시킨다(마스코트·한 줄 멘트 수준). 화면 전체를 전통 콘셉트로 덮지 않는다.
- 판단이 애매하면 "이게 요즘 잘 만든 앱처럼 보이나?"를 기준으로 삼는다. "전통적으로 보이나?"가 아니다.

## 1. 색상 토큰

```
/* 브랜드 (장금이 세이지 그린) */
--jg-primary:        #5C6E3D;   /* 메인 포인트 (액션, 강조) */
--jg-primary-strong: #47562F;   /* 눌림·진한 강조 */
--jg-primary-soft:   #EEF1E6;   /* 포인트 배경 틴트 */

/* 배경 (거의 흰색 — 시안 기준. 아이보리 아님) */
--jg-bg:      #FBFBFA;   /* 앱/페이지 배경 — 순백에 가까운 오프화이트 */
--jg-surface: #FFFFFF;   /* 카드·시트 표면 (흰색) */
--jg-surface-sunken: #F4F4F2; /* 입력·눌린 영역 (옅은 회색) */

/* 텍스트 (고대비, 토스식 위계) */
--jg-ink:     #1F1D18;   /* 제목·본문 (거의 검정, 따뜻한 톤) */
--jg-ink-sub: #6B675E;   /* 보조 텍스트 */
--jg-ink-mute:#9B978C;   /* 캡션·플레이스홀더 */

/* 경계·구분 */
--jg-line:    #E7E4DC;   /* 카드 테두리·구분선 */

/* 영역 액센트 (구매/보관/처리 — design-notes 성격과 매칭) */
--jg-buy:  #1F804D;  --jg-buy-bg:  #E9F6EE;   /* 구매 · 정답형 */
--jg-keep: #2F6E86;  --jg-keep-bg: #E8F2F6;   /* 보관 · 합의형 (차분한 청록) */
--jg-use:  #C4712A;  --jg-use-bg:  #FBF0E4;   /* 처리 · 확산형 (따뜻한 주황) */

/* 상태 */
--jg-star:    #F0A93B;   /* 별점 */
--jg-danger:  #C1453B;   /* 경고·삭제 */
```

색은 반드시 위 토큰에서 고른다. 형광·고채도 원색 금지. 영역 액센트는 브랜드 그린과 조화되는 저채도로 유지.

## 2. 타이포그래피

- 폰트: 시스템 산세리프 스택 (`-apple-system, "Apple SD Gothic Neo", "Pretendard", "Malgun Gothic", sans-serif`). 별도 폰트 로드 없이 어디서나 일관.
- 위계 (토스식: 크고 굵은 제목 + 확실한 대비):

| 역할 | 크기 | 굵기 | 색 |
|------|------|------|-----|
| 히어로 타이틀 | 30~34px | 800 | ink |
| 화면 제목 (H1) | 24px | 800 | ink |
| 섹션 제목 (H2) | 19px | 700 | ink |
| 카드 제목 | 16px | 700 | ink |
| 본문 | 15px | 400 | ink |
| 보조/캡션 | 13px | 500 | ink-sub / ink-mute |

- 색으로 위계를 만들지 말고 **크기·굵기**로 만든다. 회색은 보조 텍스트에만.

## 3. 간격 · 모서리 · 그림자 (4px 그리드)

```
간격 스케일: 4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 (px)
카드 안쪽 패딩: 16~20px
카드 간 간격: 12~16px
화면 좌우 여백: 20px

모서리 반경:
--r-sm: 10px  (칩·작은 요소)
--r-md: 16px  (카드·입력)
--r-lg: 20px  (큰 카드·시트)
--r-pill: 999px (필터칩·태그·둥근 버튼)

그림자 (옅게):
--shadow-card: 0 2px 10px rgba(31,29,24,.05);
--shadow-float: 0 6px 20px rgba(31,29,24,.10);  /* 시트·플로팅 */
```

여백을 아끼지 않는다. 토스식으로 요소 사이 숨 쉴 공간을 준다.

## 4. 핵심 컴포넌트

- **버튼**
  - Primary: 배경 `--jg-primary`, 흰 글자, 반경 `--r-md`(주요) 또는 pill, 세로 패딩 14px+, 굵기 700. 화면당 주요 버튼 1개.
  - Secondary: 배경 흰색/틴트, `--jg-primary` 글자, 얇은 line 테두리.
  - 최소 탭 높이 44px.
- **카드**: `--jg-surface` 배경, `--r-lg` 반경, `--shadow-card`, `--jg-line` 테두리(선택). 안쪽 16~20px.
- **필터칩**: pill 모양. 선택 시 `--jg-primary` 채움 + 흰 글자, 미선택 시 틴트 배경 + 포인트 글자. **보관 영역은 재료 타입에 따라 칩 셋이 달라진다**(design-notes 참고).
- **리스트/방법 카드**: 좌측 썸네일(또는 색 블록) + 제목 + 별점(`--jg-star`)·평가수 + 짧은 요약 2줄. 순위/베스트 뱃지는 좌상단.
- **검색바**: 상단 고정, `--jg-surface-sunken` 배경, pill 또는 `--r-md`, 돋보기 아이콘 + 플레이스홀더("재료를 검색해보세요"). 토스처럼 검색을 진입점으로.
- **하단 탭바 / 액션**: 주요 이동은 하단 탭, 작성 등 생성 액션은 눈에 띄는 플로팅 또는 채움 버튼.

## 5. 페이지 레이아웃 골격 (⚠️ 모든 페이지가 반드시 따른다)

프로토타입·목업을 만들 때 레이아웃을 즉흥적으로 짜지 않는다. **아래 골격을 그대로 쓴다.** 이게 팀 시안의 뼈대다.

### 데스크톱 (기본 폭 1440px, 콘텐츠 max-width ~1240px)

```
┌────────────────────────────────────────────────────────┐
│ 글로벌 헤더 (sticky)                                      │
│  [로고+태그] ── 검색바(중앙, 넓게) ──  🔍 🔔 [로그인]        │
├──────────┬──────────────────────────────┬──────────────┤
│ 좌 사이드바 │  중앙 본문 (메인 콘텐츠)         │  우 사이드 패널 │
│  240px    │  flex, 폭 채움                 │  300~320px    │
│  sticky   │                              │  sticky       │
│           │                              │               │
│ · 홈       │  [브레드크럼]                  │ (페이지별 요약/ │
│ · 재료 허브 │  [페이지 콘텐츠]                │  랭킹/노트 카드)│
│ · 구매 가이드│                              │               │
│ · 보관 가이드│                              │               │
│ · 처리 레시피│                              │               │
│ · 오늘의 추천│                              │               │
│ · 커뮤니티  │                              │               │
│ ─────────  │                              │               │
│ 최근 본 재료│                              │               │
│ (아이콘 목록)│                              │               │
│ ─────────  │                              │               │
│ [프로모 카드]│                              │               │
│ "재료 고민이 │                              │               │
│  끝!"      │                              │               │
└──────────┴──────────────────────────────┴──────────────┘
```

- **좌 사이드바(고정 필수):** 아이콘+2줄(제목+설명) 네비 항목. 현재 페이지는 초록 배경 하이라이트. 아래에 "최근 본 재료"(작은 아이콘 리스트) + "재료 고민이 끝!" 프로모 카드(장금이 일러스트). `position: sticky`.
- **우 사이드 패널(페이지별 내용, 폭은 고정):** 재료 허브/구매/보관 페이지에 요약·랭킹·노트 카드. 예) "장금이 요약 노트", "인기 보관법 TOP 3", "지금 이 마트 수박은?", "함께 보면 좋은 콘텐츠". `position: sticky`.
- **중앙:** 브레드크럼 → 재료 히어로(해당 시) → 구매/보관/처리 탭(재료 상세 계열) → 본문.
- **홈(`/`)은 예외:** 좌 사이드바 없이 풀폭 히어로 + 검색 중심(시안 6). 홈만 마케팅 랜딩 성격이라 사이드바를 쓰지 않는다.

### 모바일 (폰 프레임)

```
┌─────────────────┐
│ 상단 앱바         │  ← 뒤로 · 제목(가운데) · ♡ ↑   (상세)
│                 │  또는  로고 · 🔔 👤            (홈/허브)
├─────────────────┤
│ [구매][보관][처리] │  ← 재료 상세 계열은 세그먼트 탭 (현재만 초록 언더라인)
├─────────────────┤
│  세로 스택 콘텐츠  │
│                 │
├─────────────────┤
│ 하단 탭바         │  홈 · 재료 · [장금이 FAB] · 커뮤니티 · 마이페이지
└─────────────────┘
```

- 하단 탭바 5칸, 가운데는 **장금이 FAB**(원형, 살짝 위로 튀어나옴).
- 데스크톱의 좌 사이드바 = 모바일에선 하단 탭바 + "재료" 탭으로 대체.
- 데스크톱의 우 요약 패널 = 모바일에선 본문 하단에 세로로 편입.

### 두 플랫폼 공통

- 콘텐츠는 카드로 묶고, 카드는 `--r-lg`·`--shadow-card`·흰 배경.
- 절대 상단 가로 메뉴만으로 데스크톱을 만들지 않는다(좌 사이드바가 기본). 홈만 예외.

## 6. 영역별 화면 뼈대 (구매/보관/처리)

`design-notes.md`의 정보 성격에 따라 **화면 구조를 다르게** 잡는다. 같은 카드 3개로 만들지 않는다.

- **구매(정답형)** → 정독형 가이드. 기준 체크리스트 카드 + 현장 데이터 배지. 읽기 중심, 평점 최소.
- **보관(합의형)** → 필터+정렬 비교 뷰. 재료별 동적 필터칩 + 방법 카드 별점 정렬 + 베스트 뱃지.
- **처리(확산형)** → 피드형 UGC. 작성 버튼 상단, 유저 콘텐츠 카드 스트림, 참여 보상(가챠/도감).

## 7. Figma 오토레이아웃 매핑 (도구: Figma)

디자인은 Figma로 구현한다. HTML 목업을 만들 때부터 **오토레이아웃으로 옮기기 쉽게** 구성한다.

- HTML의 세로 스택(`flex-direction: column`) → Figma 세로 오토레이아웃 프레임. 가로 배치 → 가로 오토레이아웃.
- `gap` = 오토레이아웃 `itemSpacing`. `padding` = 오토레이아웃 패딩. 위 4px 그리드 값을 그대로 쓴다.
- 카드는 "hug contents"(자식에 맞춤), 화면 컨테이너는 "fill container"(폭 채움). 목업 주석에 이 의도를 남긴다.
- 텍스트는 고정폭 금지 → 컨테이너 폭 안에서 wrap(오토레이아웃 `fill` + text `HEIGHT` auto-resize)에 대응.
- 반복 요소(방법 카드, 필터칩)는 **컴포넌트+변형** 후보로 표시한다(예: 카드에 `<!-- component: MethodCard -->` 주석).
- 색·반경·간격은 위 토큰을 **디자인 변수/스타일**로 등록할 것을 전제로 이름을 붙인다.

Figma MCP(`use_figma`) 사용 시 오토레이아웃·변수 문법은 figma-use 스킬을 따른다. MCP 호출 한도가 있을 수 있으니, 먼저 HTML 목업으로 확정한 뒤 Figma로 옮긴다.

## 8. 레퍼런스 목업 & 확정 컴포넌트 패턴 (⚠️ 재료 상세 화면은 이걸 따른다)

**기준 예시:** [`docs/mockups/ingredient-detail-mockup.html`](../../../docs/mockups/ingredient-detail-mockup.html) — 수박 상세(구매·보관·처리 3탭)를 여러 차례 논의로 다듬어 확정한 목업. 재료 상세 계열 화면을 만들 땐 **이 파일을 열어 CSS·구조를 재사용**한다. 아래는 그 목업에서 확정된 패턴 요약(세부 구현은 파일 참조).

### 8-1. 히어로 = 장금이 화자 + SEO h1 (세 탭 공통)
- 카피 위에 **작은 화자 라벨**(👩‍🍳 장금이 왈,) — 아바타 22px, 카피와 경쟁시키지 않음. 큰 삽화·수박 이모지를 카피 옆에 두지 않는다(카피 폭을 먹어 줄바꿈 깨짐).
- **큰 몰입 카피**가 주인공: "좋은 수박은 이렇게 고르는 것이옵니다" / "수박은 이렇게 보관하는 것이옵니다" / "남은 수박으로 무엇이든 지어보아요". 장금이 사극 톤은 **카피에서만** 가볍게.
- 그 위에 **작은 h1**(`수박 고르는 법`)을 별도로 둔다 → 화면엔 작게, 문서 구조상 h1(SEO 검색어). 몰입 카피와 SEO를 **역할 분리**.

### 8-2. 구매 — 확률 게이지 + 기준 카드
- **확률 게이지(상단 sticky):** 기준을 체크할수록 "이 재료가 좋을 확률"이 실시간 상승. 스크롤해도 상단 고정.
  - **색은 상태 기반:** 0%=중립 회색 → ~33% 주의(황) → 그 이상 초록. **재료 무관**(수박·양파 다 같은 구매색, 재료색 아님).
- **기준 카드 = 2열 편집 디자인:** 카드 테두리 없이 **얇은 십자 구분선**으로 나눔(그리드 컨테이너만 카드). **번호**("1. 배꼽 확인") + 텍스트 좌측 + **이미지 우하단 뒤 레이어**. 텍스트는 앞 레이어 + **흰색 후광(text-shadow halo)**으로 이미지 위에서도 또렷. 행 높이 통일.
  - **선택 = 은은한 초록 배경**만(동그라미 체크박스·별도 배지 없음). 제목 앞에 ✓를 붙여 텍스트를 밀지 않는다.
- **Best/Worst 대비 헤더:** 👍/👎 라벨(색은 여기만), **이미지 배경은 무채색**(사진 선명). 카피는 **재료 무관**("이런 걸 고르세요/이건 피하세요") — 재료명 하드코딩 금지.
- **출처(신뢰 = SEO·GEO 핵심):** 무채색 배경 + 행 전체 토글(`<details>`). 요약 문구 항상 노출, 상세만 접기. criteria별 유니크 출처 집계, 등급(공식 ✔ / 영상 ▶) 구분. 접어도 SSR로 DOM에 존재해야 색인됨.

### 8-3. 보관 — 방법 비교 카드
- **방법 비교가 주인공.** 썸네일 크게(72px, 사진으로 방법 인지), **순위는 썸네일 좌상단 오버레이**(별도 컬럼 없이 텍스트 폭 확보). 1위 금색 배지.
- **필터 = 데이터 3축 드롭다운**(`storage.filters`의 form/place/period와 1:1). 데이터에 없는 필터('선호도' 등) 금지. 각 방법 `tags`로 필터링. 필터 5개까지 2줄, 6개+ 혼합/바텀시트.
- **태그 무채색**(영역색으로 튀지 않음). 별점은 목업 임시값 OK.
- **신뢰 = 장금이 코멘트**(사용자 후기는 로그인·UGC 후속이라 지금은 대체). 카드 **하단 전체 폭 한 줄**(좁은 컬럼에 넣으면 빨리 잘림). 큰 박스로 시선 뺏지 말 것 — 보조 한 줄.

### 8-4. 처리 — 활용/폐기 피드
- 상단 **활용 / 폐기 세그먼트**로 갈림. 활용=레시피 UGC 피드, 폐기=분리배출 안내(`handling.dispose`).
- 활용은 **정렬만**("인기순"), 카테고리 필터 없음(확산형은 피드 탐색이 자연스러움). 하단 **작성 CTA** 고정.

### 8-5. 기획 명세 패널 (`spec-*`) — 목업에 명세를 나란히
목업은 화면만 그리지 않고, **우측에 기획 명세 패널을 함께 둔다.** 재사용 클래스:
- `.spec-layout` (좌 목업 / 우 명세 flex, `justify-content: space-between`으로 패널을 **우측 끝**에 붙임)
- `.spec-stage` (폰 목업들, 폰 2개폭 상한 두고 3번째는 아랫줄 wrap)
- `.spec-panel` (우측, `position: sticky` + 내부만 상하 스크롤, **절대 상단으로 올리지 않음**)
- `.spec-head` (패널 전체 헤더 "📋 기획 명세") + `.spec-card`(블록: 데이터 매핑·라우팅 / 탭별 기능 요구사항 / 확정 사항)
- 패널엔 **데이터 매핑(어떤 schema 필드를 쓰는지)·라우팅·요구사항·확정 사항**을 표로. 논의로 결정이 바뀌면 **패널을 즉시 싱크**한다(목업과 명세가 어긋나면 안 됨).

### 8-6. 논의로 목업을 구체화하는 워크플로 (필수)
목업은 한 번에 완성되지 않는다. **질문 → 결정 → 반영 → 검증**을 반복한다:
1. 애매한 선택은 **추천안을 제시하고 사용자에게 확정**받는다(임의 결정 금지). "토스였어도 이렇게 했을까?"를 판단 기준으로.
2. **참조 이미지가 있으면 그것이 스펙**이다. 이미지에 없는 요소(필터·카피·필드)를 추측으로 넣지 않는다.
3. 결정이 나면 목업에 반영하고, **Playwright로 실제 렌더를 스크린샷/측정해 검증**한다(줄바꿈·겹침·정렬·넘침은 눈으로 확인). 로컬 Playwright는 스크래치패드에 격리 설치해 프로젝트를 오염시키지 않는다.
4. 논의 결과를 **우측 명세 패널에 싱크**하고, 재사용할 패턴이면 **이 문서(design-system.md)에 추가**한다.

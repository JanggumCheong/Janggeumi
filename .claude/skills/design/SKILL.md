# design — 화면 목업 생성 스킬

장금이의 화면을 **디자인 시스템에 맞춰** HTML 목업으로 만든다. 매번 스타일을 새로 지어내지 않고 정의된 토큰·규칙을 재사용해 일관성을 지킨다.

## 0. 대상 화면

디자인할 화면: **$ARGUMENTS**

인자가 비어 있으면 "어떤 화면을 목업으로 만들까요?"를 먼저 묻는다.

## 1. 기준 문서 먼저 읽기 (필수)

목업을 만들기 전에 **반드시** 두 문서를 읽는다:

1. `docs/foundation.md` — 프로젝트 방향·톤. 세부가 어긋나면 이 파일이 우선.
2. [design-system.md](design-system.md) — 색·타이포·간격·컴포넌트·영역별 화면 뼈대·Figma 오토레이아웃 규칙. **여기 정의된 토큰만 사용한다.**

## 2. 핵심 원칙 (요약 — 상세는 design-system.md)

- **현대적으로.** 장금이는 매개체일 뿐, 디자인은 전통/사극 느낌 금지. "요즘 잘 만든 앱처럼 보이나?"가 기준.
- **토스식 간결함.** 한 화면 한 할 일 · 넉넉한 여백 · 큰 타이포 위계 · 절제된 색 · 큰 라운드 카드. 판단이 애매하면 **"토스였어도 이렇게 했을까?"**로 결정.
- **영역별로 화면 구조가 다르다.** 구매=정독형 가이드 / 보관=필터+정렬 비교 / 처리=피드형 UGC. 같은 카드 3개로 만들지 않는다.
- **토큰만 사용.** 색·간격·반경·폰트는 design-system.md의 값에서만 고른다. 임의 색 금지.
- **장금이 화자 톤 통일.** 세 탭 히어로 모두 "👩‍🍳 장금이 왈, ~것이옵니다" 화자 + SEO용 h1 분리(8-1).
- **데이터에 없는 UI 금지.** 필터·필드·카피를 추측으로 넣지 않는다. 참조 이미지가 있으면 그것이 스펙. 재료명 하드코딩 금지(다른 재료도 쓴다).
- **논의로 구체화.** 애매한 선택은 추천안 제시 후 사용자 확정. 결정은 목업 반영 → Playwright 검증 → 명세 패널 싱크(8-6).

## 3. 목업 생성

- **재료 상세 계열(구매·보관·처리) 화면이면** 먼저 [`docs/mockups/ingredient-detail-mockup.html`](../../../docs/mockups/ingredient-detail-mockup.html)을 연다. 이미 논의로 확정된 레퍼런스라 CSS·구조를 그대로 재사용한다(design-system.md 8장 참조). 새 화면이면 [mockup.html](mockup.html) 템플릿으로 시작.
- `<style>`의 CSS 변수(디자인 토큰)와 컴포넌트 클래스를 그대로 쓰고, 화면 내용만 채운다.
- 모바일 앱 화면이면 **일반적인 폰 프레임**(폭 ~340px, 상태바 + 하단 탭바) 안에 그린다(템플릿에 `.phone` 제공).
- 여러 화면을 보여줄 때는 가로로 나란히(진입점 → 상세). **우측엔 기획 명세 패널(`spec-*`)을 함께 둔다**(design-system.md 8-5).
- 결과물은 `docs/mockups/<화면명>-mockup.html`로 저장.
- **Playwright로 렌더를 검증**한다(스크래치패드에 격리 설치). 줄바꿈·겹침·정렬·넘침은 코드가 아니라 실제 스크린샷으로 확인.

## 4. Figma 오토레이아웃을 염두에 둔 구조 (도구: Figma)

디자인은 Figma로 구현하므로, HTML을 **오토레이아웃으로 옮기기 쉽게** 만든다:

- 세로 스택 = `flex-direction: column` → 세로 오토레이아웃. 가로 = 가로 오토레이아웃.
- `gap`/`padding`은 4px 그리드 토큰 값으로. 오토레이아웃 itemSpacing/padding에 그대로 매핑.
- 반복 요소(방법 카드·필터칩·진입 카드)에는 `<!-- component: 이름 -->` 주석을 달아 컴포넌트+변형 후보임을 표시.
- 화면 컨테이너=fill, 카드=hug 의도를 주석으로 남긴다.

Figma에 실제 반영할 때는 figma-use 스킬로 오토레이아웃·변수를 만든다. Figma MCP 호출 한도가 있을 수 있으니 **HTML 목업으로 먼저 확정**한 뒤 옮긴다.

## 5. 마무리

- 저장한 파일 경로를 알린다.

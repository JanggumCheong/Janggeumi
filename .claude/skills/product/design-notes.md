# 영역별 정보 성격 (기획 참고)

장금이는 식재료의 **구매 · 보관 · 처리** 세 영역을 다룬다. 세 영역은 카테고리만 다른 게 아니라 **정보의 성격 자체가 다르다.** 기능을 기획할 때 어느 영역인지 먼저 판단하고, 그 성격에 맞는 UI·데이터·사용자 행동을 잡는다.

판단 기준: **"이 정보에 정답이 있는가, 있다면 누가 정하는가?"**

## 🛒 구매 — 정답형 (객관적)

- **정보 성격:** 정답이 수렴한다. "좋은 파 고르는 법"은 사람마다 다르지 않다.
- **정답의 주인:** 전문가 / 데이터
- **필요한 힘:** 검증된 기준 (권위). 평점으로 흔들면 오히려 신뢰가 떨어진다 → 집단지성보다 기준 제시.
- **사용자 행동:** 읽기 중심 (정독형 가이드).
- **두 층:** 불변 기준(교과서) + 현장 변수(지점별 실시간 데이터, 예: "이 마트 수박 89% 확률로 맛있다").
- **UI 방향:** 기준 체크리스트, 큰 가이드 카드, 데이터 배지.

## 🧊 보관 — 합의형 (조건부·선호)

- **정보 성격:** 정답은 없지만 "더 나은 합의"는 있다.
- **정답의 주인:** 상황 + 집단
- **필요한 힘:** 필터 + 집단지성 (합의). 평가·후기로 베스트가 위로 부상.
- **사용자 행동:** 읽기 + 평가 (비교·정렬 뷰).
- **핵심:** **재료마다 필터 축이 다르다.** 파는 냉장/손질/신선도, 감자는 상온/광차단. 필터를 전역 고정하면 안 되고 재료 타입별로 필터 셋이 달라야 한다.
- **UI 방향:** 재료별 동적 필터칩, 방법 카드 + 별점/평가수 정렬, "베스트" 뱃지.

## 🍳 처리 — 확산형 (창발적)

- **정보 성격:** 정답이 가장 없고 가능성이 무한. 파 1대로 파전/파기름/파김치…
- **정답의 주인:** 사용자
- **필요한 힘:** UGC / 참여 보상 (다양성). 계속 새 콘텐츠가 올라와야 살아있다.
- **사용자 행동:** 쓰기 중심 (피드형 UGC).
- **UI 방향:** 유저 콘텐츠 피드, 작성 버튼, 참여 보상.

## 성격 스펙트럼

```
구매  ───────────  보관  ───────────  처리
정답형              합의형              확산형
읽기 중심           읽기 + 평가          쓰기 중심
권위 / 데이터        필터 + 집단지성      UGC / 참여보상
장금이의 비법서      장금이의 손맛        다 같이 짓는 수라간
```

**설계적 함의:** 세 영역을 "같은 카드 UI 3개"로 만들면 안 된다. 구매=정독형 가이드, 보관=필터+정렬 비교 뷰, 처리=피드형 UGC 스트림.

## 확정된 영역별 UI (재료 상세 화면)

`docs/mockups/ingredient-detail-mockup.html`에서 논의로 확정. 세부 구현·CSS는 design-system.md 8장.

- **구매(정답형):** Best/Worst 대비 헤더(이미지로 눈 학습) + **확률 게이지**(기준 체크 시 "좋을 확률" 상승, 상단 sticky, 0%→중립·초록으로 상태색) + 2열 기준 체크 카드 + **출처**(신뢰=SEO/GEO 핵심, 등급 구분·요약 노출·상세 접기). 재료명 하드코딩 금지(카피 재료 무관).
- **보관(합의형):** 데이터 3축(`storage.filters`) 드롭다운 필터 + 방법 카드(썸네일 크게, 순위 오버레이, 무채색 태그) 별점 정렬. 후기는 로그인·UGC 후속 → 지금은 **장금이 코멘트**(검증된 큐레이션)로 대체. 방법 비교가 주인공, 코멘트는 하단 한 줄 보조.
- **처리(확산형):** 활용/폐기 세그먼트. 활용=레시피 UGC 피드(**카테고리 필터 없이 정렬만** — 피드 탐색) + 작성 CTA. 폐기=분리배출 안내(`handling.dispose`).

## 후속 기능 의존 요소 (지금은 대체)

로그인·UGC가 있어야 하는 것들은 지금 대체안으로 기획하고, 후속임을 명시한다:
- 사용자 후기/평점 → **장금이 코멘트**(우리가 쓰는 검증된 큐레이션). 별점은 목업 임시값.
- "지금 이 마트 89%" 같은 현장 데이터 → 실시간 파이프라인 후속. 지금은 확률 게이지(사용자가 직접 체크)로 대체.

섹션별 기능 요구사항

  • 검색 = 모드 전환 — 빈 검색창은 탐색 모드(🔥·🌱+칩). 1글자+ 입력 시 검색 모드로 전환: 맥락 섹션 전부 숨기고 검색창 바로 아래 행(리스트) 자동완성. 입력과 결과가 멀어지지 않게 코앞에 붙임.
  • 검색 매칭 — name+aliases 부분일치, 접두 일치 우선. 별칭 매칭이면 이름 뒤 흐린 별칭(대파 · 파). 0건 → 빈 상태 + 칩 유도. 1건이면 Enter로 바로 상세. X로 탐색 모드 복귀.
  • 카테고리 칩 = 브라우징 축 — categories와 1:1(하드코딩 금지). 전체: A-Z 나열 없이 "🥕 카테고리를 골라 둘러보세요" 안내. 채소·과일 등: 그 자리에 그 카테고리 전체 그리드(가나다) 펼침. 검색어와 AND.
  • 🔥 많이 찾는 — viewCount 내림차순 상위 6. 순위 오버레이. 별점 없음. 위에 장금이 화자 한 줄.
  • 🌱 제철 — 오늘 월 ∈ seasonMonths. 0건이면 섹션 숨김. 위에 장금이 화자 한 줄.
  • 카테고리 그리드 — 칩 선택 시만 등장. 가나다순(완결 목록). planned는 "준비 중" 배지 + 진입 차단.
  • 장금이 화자 — 캐릭터 일러스트 대신 말투로 스밈. 작은 아바타 1개 + 🔥/🌱 큐레이션 멘트 + 안내·0건 카피를 사극 톤으로. 🔥·🌱가 "데이터"가 아니라 "장금이의 추천"으로 읽혀 신뢰(믿음) 톤 강화.

확정 사항

  • 정렬 탭 폐기 → 발견=🔥🌱 / 브라우징=칩 / 특정=검색 세 갈래. 정렬 축 고르는 인지 부담 제거.
  • A-Z 전체 나열 폐기 — 가나다 미리보기는 "앞 글자(ㄱ~ㄷ)만 항상 보임" + 카탈로그 커지면 무한 스크롤. 전체 나열 자체를 없애 12개든 200개든 ① 길이 불변.
  • 가나다순은 "카테고리 완결 목록"에서만 — 발견용 미리보기엔 안 씀(색인 순서라 발견에 부적합).
  • 검색은 모드 전환 — 입력 시 맥락 섹션 숨기고 행 자동완성. 검색과 결과가 멀어지는 문제 해소(탐색↔검색 두 화면이 공존하지 않음).
  • 장금이는 화자로 스밈 — 기능성 화면이라 캐릭터 크게 X. 문구 + 작은 아바타 1개로.
  • 재료별 별점 미채택 — 평점은 storage.methods[].rating이 원천, 재료 단위 근거 없음.
  • viewCount 필드 신설 — 조회수는 탐색 허브에서 생기는 실제 집단 신호. 초기 큐레이션 시드 → 후속 실집계.
미채택: 인기순 탭(산정X) · 최신순(죽은 축) · 재료별 별점 · 선호도 필터

SEO / GEO

  • h1 = 식재료 검색 (visible), 서브 "모든 식재료를 한눈에".
  • 모든 재료명 실제 텍스트 SSR. 🔥·🌱·카테고리 그리드 전부 서버 렌더 DOM에 존재, 검색·필터는 클라이언트 토글(표시만).
  • JSON-LD ItemList — 각 ListItem = name + /ingredients/{id}.

범위 밖 / 후속

  • viewCount 실집계 파이프라인(GA/서버 로그) — 지금은 시드값.
  • 초성 검색·오타 보정 — 1차는 부분일치.
  • 최근 본 재료(홈과 저장소 공유) 포함 여부 — 팀 확인.
  • 재료별 별점·리뷰(UGC) / 로그인·마이페이지 플로우.

# 검색(재료 탐색 허브) 페이지 기획

> 라우트 `/search`. 재료를 **검색**하거나 **맥락 섹션(🔥 많이 찾는·🌱 제철)으로 발견**하고 **카테고리 칩으로 브라우징**해 재료 상세(`/ingredients/[slug]`)로 진입시키는 모바일 허브 화면.

## 0. IA 변경 (2026-07-08) — 단일 검색 페이지 + 상태 전환

허브/전용 검색을 잠시 분리했다가 **다시 단일 페이지로 합쳤다.** 허브가 실질적으로 검색 페이지의 껍데기라 분리 이득이 없었다. `/search` 한 화면 안에서 **상태 전환**으로 처리한다.

- **`/search` 단일 페이지** — 편집형 검색창 하나. **검색은 "입력"이 아니라 "실행(제출)"이 기준.**
  - **미검색·타이핑 중**: 🔥·🌱 캐러셀 + 카테고리 칩 + 안내(발견·제안 상태). 글자를 쳐도 화면은 바뀌지 않는다.
  - **검색 실행(Enter 또는 돋보기 클릭)**: `submittedQuery` 확정 → 비동기 요청(react-query) → **로딩 스켈레톤 → 정렬 탭 + 결과 그리드.** (7-1의 "리스트 자동완성"을 그리드+정렬 탭 결과로 대체.)
  - 입력을 비우면(검색창 X/전체 삭제) 발견 모드로 복귀.
  - **데이터 소스**: 현재는 로컬 목(`features/search/api.ts`의 `searchIngredients`)을 Promise로 감싼 시드. 백엔드 검색 GET 엔드포인트(예: `/v1/search?q=`)가 생기면 그 함수 본문만 fetch로 교체(호출부·상태 모델 불변). 정렬·카테고리는 `queryKey`에 포함돼 재검색 없이 갱신.
- **헤더 검색 버튼** — 검색 영역(`/search`) 밖에서 🔍 노출 → `/search` 진입.
- **정렬 탭(`SearchSortTabs`) 부활** — 6장에서 폐기했으나 결과 화면에 인기순/가나다순 축으로 유지(팀 결정, 2026-07-08). 6장의 폐기 서술은 이 결정으로 갱신됨.

## 1. 한 줄 정의

`/search`는 재료명을 아는 사람은 **검색**으로, 모르는 사람은 **🔥·🌱 큐레이션 섹션과 카테고리 칩**으로 훑어 원하는 재료 상세로 최단 진입하게 하는 식재료 탐색 허브다.

## 2. 문제 (Why)

- 홈(`/`)은 "인기 상승 12개 + 최근 본"만 보여줘 **전체 카탈로그를 훑을 진입점이 없다.** 원하는 재료가 12개 밖이면 검색밖에 길이 없다.
- 초보 사용자는 "지금 뭘 사야 하지"를 재료명으로 특정하지 못한다 → **제철·인기로 발견**하고 **카테고리로 브라우징**할 창구가 필요하다.
- 검색 진입·AI 인용이 성장 동력인데, 재료 목록이 텍스트로 색인되는 **탐색 허브 h1 페이지**가 없다.

## 3. 대상 & 시나리오 (Who/When)

- **탐색형 초보** — "오늘 장 볼 건데 뭐 사지?" → `/search` 진입 → 스크롤만 → `🔥 지금 많이 찾는` / `🌱 지금 제철` 섹션에서 눈에 띄는 재료 탭 → 상세. **정렬 축을 고를 필요 없이** 이미 큐레이션된 걸 만난다.
- **목적형 사용자** — "양파 보관법 보러 왔다" → 검색창에 `양파` → 실시간 매칭 → 상세.
- **카테고리 브라우징** — "채소만 볼래" → `채소` 칩 → 그 자리에 채소 전체 그리드가 가나다순으로 펼쳐짐.

## 4. 핵심 가치 매핑

- **빠른 확신** — 검색 1회 또는 스크롤/칩 1탭으로 상세까지. 카드는 "이름·카테고리·제철"만 담아 결정에 필요한 것만.
- **믿음(집단지성)** — `🔥 많이 찾는`은 **조회수(`viewCount`)라는 실제 집단 신호**로 큐레이션. 장금이 화자 멘트를 얹어 "데이터"가 아니라 "장금이의 추천"으로 읽히게. 근거 없는 지표(재료별 별점)는 넣지 않아 신뢰를 오염시키지 않는다.
- **끝까지** — 제철 섹션으로 "지금 사면 좋은 것"을 놓치지 않게.

## 5. 영역 & 정보 성격

**구매·보관·처리 어디에도 속하지 않는 "탐색(navigation) 계층".** 세 영역의 정보를 담지 않고 **그 입구만 연다.**

- 재료 단위 지표를 함부로 만들지 않는다. 평점은 `storage.methods[].rating`(합의형·보관법별)이 원천 → **재료 카드에 별점 없음.** (참조 이미지의 ★4.7 미채택 — 근거 없음.)
- **단, 조회수는 탐색 허브에서 자연히 생기는 집단 신호**라 재료 단위 지표로 정당하다 → `🔥 많이 찾는` 큐레이션 근거로 채택.

## 6. 화면 구조 — 왜 "정렬 탭·전체 나열"이 아니라 "발견·브라우징·검색 세 갈래"인가

토스식 판단: 12~20개 목록에 `인기순/가나다순/제철순` **정렬 탭**을 다는 건 엑셀 정렬이지 서비스가 아니다. 사용자에게 "정렬 축을 고르는 인지 부담"을 지운다. 대신 **사용자가 실제 던지는 질문**을 구조로 세운다.

- 폐기: **정렬 탭 전체**, 특히 `최신순`(재료 탐색에서 아무도 원하지 않는 죽은 축 — `order`를 억지로 쓴 것).
- 폐기: **① 기본의 A-Z 가나다 미리보기.** 미리보기를 가나다로 자르면 앞 글자(ㄱ~ㄷ)만 영원히 노출되고 뒤 재료는 안 뜬다 → "보이는 것만 보인다". 미리보기는 발견 목적인데 가나다는 색인 순서라 목적 불일치. 게다가 🔥·🌱가 이미 발견을 맡아 A-Z 미리보기는 군더더기. 카탈로그가 12개→200개로 커지면 무한 스크롤로 무너진다.
- 채택: 세로 **맥락 섹션 스택** + **칩이 브라우징 축.**

**세 갈래 역할 분담:** 발견 = 🔥·🌱 / 브라우징 = 카테고리 칩 / 특정 재료 = 검색.

```
검색창 (일반 배치, sticky X)
카테고리 칩 (가로 스크롤)        ← A-Z 영역의 스위치
─────────────
🔥 지금 많이 찾는 재료   (가로 스크롤 카드, 장금이 화자 멘트)
🌱 지금 제철이에요        (가로 스크롤 카드, 장금이 화자 멘트)
─────────────
[전체 칩]  → "🥕 카테고리를 골라 둘러보세요" 안내만 (A-Z 나열 X)
[채소 칩]  → 채소 전체 그리드(가나다) — 완결 목록이라 가나다 자연스러움
```

이러면 카탈로그가 12개든 200개든 ① 기본 화면 길이가 안 변한다(전체 나열이 없으므로). 가나다순은 "카테고리를 고른 뒤 완결된 목록"에서만 쓴다.

## 7. 기능 명세 (행동 → 반응)

### 7-1. 검색창 + 검색 모드 (상단 일반 배치, **sticky 아님**)

**검색은 정렬·필터가 아니라 "화면 모드 전환"이다.** 입력과 결과가 멀어지면 안 되므로, 한 글자라도 들어오면 화면 전체가 결과 화면으로 바뀌고 **검색창 바로 아래에 답이 붙는다.**

- **탐색 모드(빈 검색창, 기본)** — 🔥 · 🌱 + 카테고리 칩 브라우징.
- **검색 모드(1글자+ 입력)** — 맥락 섹션을 **전부 숨기고**, 검색창 바로 아래에 **행(리스트) 자동완성** 결과. 각 행 = 이미지 · 이름 · 카테고리 라벨 · 진입 화살표. 매칭 근거가 별칭이면 이름 뒤에 흐린 별칭 노출(예: `대파 · 파`).
  - 매칭: `name` + `aliases` 실시간 부분일치(초성·오타 보정은 후속).
  - 정렬: 접두 일치 우선 → 부분 일치. (조회수 가중은 후속.)
  - 카테고리 칩은 검색 모드에서도 유지, 검색어와 AND(그 안에서 좁힘).
- 결과 0건 → "`{검색어}`에 해당하는 재료가 없어요" 빈 상태 + 카테고리 칩으로 유도(장금이 톤).
- Enter/돋보기 → 결과 1건이면 바로 상세, 여러 건이면 리스트에 머문다. 검색창 우측 X로 탐색 모드 복귀.
- placeholder: `재료를 검색해보세요 (예: 수박, 양파)` (홈과 통일).

### 7-2. 카테고리 칩 (가로 스크롤) — **A-Z 영역의 브라우징 축**
- `전체 · 채소 · 과일 · 육류 · 수산 · 곡물·견과` — **`list.json`의 `categories`와 1:1.** 하드코딩 금지.
- `전체` 칩(기본) → A-Z 나열 없이 **"🥕 카테고리를 골라 둘러보세요" 안내**만(칩으로 유도).
- `채소`·`과일` 등 칩 탭 → 그 자리에 **그 카테고리 전체 그리드**를 가나다순으로 펼침(완결 목록이라 가나다 자연스러움). 검색어와 AND.
- 활성 칩은 세이지 그린 배경.

### 7-3. 🔥 지금 많이 찾는 재료 (가로 스크롤 강조 카드)
- `viewCount` 내림차순 상위 N개(기본 6). 카드: 이미지(이모지 폴백) · 이름 · (제철이면) 제철 배지.
- **별점 없음.** 순위 숫자 오버레이(1·2·3…)로 "많이 찾는" 맥락만 전달.
- 위에 장금이 화자 한 줄("👩‍🍳 요즘 다들 이것들을 찾사옵니다").
- 검색어 입력 시(검색 모드) 숨김.

### 7-4. 🌱 지금 제철이에요 (가로 스크롤 강조 카드)
- 오늘(`currentDate`) 월이 `seasonMonths`에 포함된 재료. 카드: 이미지 · 이름 · `제철 N~M월` 배지.
- 위에 장금이 화자 한 줄("👩‍🍳 지금이 딱 제철이옵니다").
- 제철 재료 0건이면 섹션 숨김. 검색 모드 시 숨김.

### 7-5. 카테고리 그리드 (칩 선택 시)
- `전체`가 아닌 칩 선택 시에만 등장. 4열 그리드, **가나다(`name` ko locale) 정렬**(그 카테고리 완결 목록).
- 카드 = 이미지 · 이름 · (제철이면) 제철 배지.
- `status: planned`(상세 미작성) 재료도 노출하되 **"준비 중" 흐린 배지**, 탭 시 "곧 찾아뵐게요"(상세 진입 차단).

### 7-6. 장금이 화자 (캐릭터가 아니라 말투로 스밈)
- 기능성 화면이라 캐릭터 일러스트를 크게 넣지 않는다. **작은 원형 아바타 1개 + 사극 톤 문구**로만.
- 지점: 🔥/🌱 섹션 위 큐레이션 멘트 · `전체` 칩 안내("👩‍🍳 무얼 찾으시옵니까?") · 검색 0건 빈 상태.
- 효과: 🔥·🌱가 "데이터 나열"이 아니라 **"장금이의 추천"**으로 읽혀 믿음(집단지성) 가치와 연결.

### 7-7. 하단 탭바
- `홈 · 검색 · 마이페이지` — **검색 활성.**

## 8. 데이터 & 필터

전부 **`content/list.json`**(목록 경량 메타)에서 온다. 상세 JSON 불필요.

| UI 요소 | schema 필드 | 비고 |
|---|---|---|
| 카드 이름 / 검색 | `name` + `aliases[]` | 부분일치 |
| 카테고리 칩·필터 | `categories[]` + item `category` | 칩=카탈로그 정의, 필터=item |
| 이미지 / 폴백 | (스프라이트) / `emoji` | 이미지 없으면 이모지 |
| 🔥 많이 찾는 | **`viewCount` (신설)** | 조회수 내림차순. 초기 시드 → 후속 실집계 |
| 🌱 제철·제철 배지 | `seasonMonths[]` | 오늘 월 포함 판정 |
| 카테고리 그리드 정렬 | `name` (ko locale) | 칩 선택 시. 정렬 탭 없음 |
| 준비 중 배지 | `status` / `hasContent` | `planned`면 진입 차단 |
| 상세 링크 | `id` | `/ingredients/{id}` |

### `viewCount` 필드 신설 (핵심 결정)
- `list.json` 아이템에 `viewCount: number` 추가. **집계 파이프라인 붙기 전까진 큐레이션 시드값**(장금이가 실검색량 감으로 정한 임시 랭킹).
- 명세 원칙과 정합: 별점(재료 단위 근거 없음)과 달리 **조회수는 탐색 허브에서 실제로 발생하는 집단 신호**라 재료 단위 지표로 정당.
- 후속: 실집계(GA/서버 로그)로 시드값 대체. `content/schema.md`·`list.json`에 필드 반영 필요.

**넣지 않는 것(근거 없음):** 재료별 별점, 리뷰 수, 선호도 필터, 최신순, 인기순 탭.

## 9. 성공 지표 (How measure)

- **검색→상세 진입율** & **탐색 경로 분포**(검색/칩/🔥/🌱 중 어디서 왔나 — 한 경로 편중이면 다른 경로 약함).
- **🔥/🌱 섹션 CTR**: 강조 카드 노출 대비 클릭.
- **카테고리 칩 사용률**: 브라우징 축이 실제로 쓰이는지.
- **0건 검색 비율**: 높으면 별칭·초성 매칭 보강.
- **SEO**: `/search`가 "식재료 검색/재료별 보관법" 쿼리로 색인·유입되는지.

## 10. SEO / GEO

- **h1**: `식재료 검색` (visible). 서브카피 "모든 식재료를 한눈에".
- 모든 재료명은 **실제 텍스트로 SSR** — 이모지·이미지에만 의존 금지.
- 🔥/🌱/카테고리 그리드 모두 서버 렌더로 DOM에 존재. 검색·필터는 클라이언트 토글(표시만 제어, 항목은 색인됨).
- **JSON-LD**: `ItemList`(재료 목록, 각 `ListItem` = name + url `/ingredients/{id}`). 출처 신뢰 모델은 상세에서.

## 11. 범위 밖 / 미정

- **`viewCount` 실집계 파이프라인** — 지금은 시드값. GA/서버 로그 연동 후속.

🥔 감자 콘텐츠 강화 — "결론"에 "왜"를 채운다

감자 상세의 껍질 처리 · 싹 · 냉장/상온 세 질문에 결론뿐 아니라 검증된 근거를 붙여, 검색·AI가 인용할 수 있는 콘텐츠로 만든다. 새 화면이 아니라 기존 3탭이 렌더하는 데이터를 채우는 작업.

대상: potato.json · 이미 published · 3탭 전체 관여

1 문제 — 답은 있는데 "왜"가 없다

사용자가 검색하는 질문현재 potato.json빈 곳
감자 껍질 일반? 음식물? dispose.peel.way 한 줄 = 음식물 왜 음식물인지 · 지역차 · 출처 없음
감자 싹 나면 어떻게 / 왜 나나 dispose.sprout 한 줄 + 빈 related 링크 "왜 싹이 나나" 콘텐츠 자체가 없음
냉장? 상온? tip·cautions에 "냉장 해로움" "왜 나쁜가" 정면 설명 부재 · 냉장 필터에 카드 없음
결론만 있고 근거가 없으면 → 사용자는 확신 못 함(빠른 확신 실패) · AI는 인용 못 함(GEO 실패). 이 서비스의 무기인 "검증된 근거"가 정작 비어 있다.

2 대상 & 시나리오

👤 누가

감자 한 봉지 사서 며칠 지나 마주치는 자취·1인 가구. "이거 버려야 하나?", "싹 났는데 먹어도 되나?", "냉장고 넣을까?"의 순간.

🍳 시나리오 A — 껍질

감자전 해먹고 껍질 한 움큼. "음식물? 일반?" → 폐기 세그먼트에서 음식물 + 왜(사료화 가능) 확인 → 확신하고 버림.

🌱 시나리오 B — 싹

사둔 감자에 싹. "왜 났지, 먹어도 되나?" → related "감자 싹, 왜 나고 먹어도 될까?"로 이동 → 도려낼지/버릴지 판단.

🧊 시나리오 C — 냉장

여름이라 냉장 고민 → 보관탭 "냉장은 오히려 해로워요" + 저온당화·아크릴아마이드 근거 → 냉암소로.

3 핵심 가치 매핑

⚡ 빠른 확신
세 질문 모두 "결론 + 왜"를 한 화면에. 껍질=음식물 배지, 냉장=❌ 명시. 망설임 제거.
🤝 믿음 (권위)
결론마다 출처(식약처·환경부·농진청). 분리배출·독성은 지어내지 않고 공식 기준만. 구매·보관=정답형이라 권위가 무기.
♻️ 끝까지 (안 버리기)
싹·초록·껍질을 "무조건 버려"가 아니라 도려내면 먹을 수 있다 / 껍질 활용부터 제시.

4 영역 & 정보 성격 — 3탭 전체를 건드린다

질문영역정보 성격반영 위치
냉장/상온 🧊 보관 합의형(필터+비교). 단 "냉장 나쁨"은 정답에 가까움 storage.methods에 냉장 카드 추가(비추천)
싹 (왜/처리) 🍳 처리 🛒 구매 싹 원인=정답형 지식, 처리=폐기 안내 dispose.sprout 상세화 + related 실콘텐츠
껍질 🍳 처리 지자체 기준 = 정답형 dispose.peel reason·regionNote·source

5 기능 명세 — 채우는 건 데이터, 화면은 기존

🧊 보관"냉장? 상온?" 정면 답

냉암소 실온 카드(추천 1위, 기존 유지)가 상단 → 그 아래 냉장 카드 신규(recommended:false).

  • 냉장 카드 요약 = "냉장은 오히려 해로워요". cautions에 저온당화 → 아크릴아마이드 근거.
  • 필터 place=냉장 눌러도 빈 결과가 아니라 "왜 안 되는지" 카드가 뜬다 — 필터가 곧 설명이 된다.
  • 필터 정합성 복구: place 옵션 3개(냉암소·냉장·냉동) : 카드 3개.
👩‍🍳 장금이 왈, 감자는 냉장고에 넣지 마시옵소서 — 전분이 당으로 변해 오히려 해로운 것이옵니다.
🍳 처리"껍질 어디에 버려요?"

peel 카드 음식물 → 탭하면 상세.

  • reason — 수분 많아 사료·퇴비화 가능 → 음식물. (대비: 양파·마늘 마른 껍질은 일반쓰레기 — 사료 부적합)
  • regionNote — 지자체 처리시설 따라 기준 상이. 애매하면 관할 확인.
  • 🏛 식약처 · 분리배출 안내
🍳 처리"싹 난 감자"

sprout 카드 음식물 → "싹·초록 도려내고, 많으면 배출".

  • reason — 싹·초록에 솔라닌·차코닌 독소 집중. 조금이면 도려내고, 많으면 폐기.
  • cautions — 도려낸 뒤에도 쓴맛 나면 먹지 말 것. 🏛 식약처
🔎 SEO/GEOrelated → "감자 싹, 왜 날까?" 실콘텐츠

빈 링크(/재료/potato/싹)를 지식 콘텐츠로 승격. h1 = 검색어형.

  • 싹 나는 이유 — 감자는 살아있는 덩이줄기 → 빛·온도·습도가 발아 촉진.
  • 솔라닌 + 안전 손질. 검색·AI 진입 콘텐츠. HowTo/FAQPage JSON-LD, 출처 SSR 색인.

6 데이터 & 필터 정합성

➕ 채울 필드

  • storage.methods[] += fridge-avoid 카드 (tags.place=냉장 → 정합 복구)
  • dispose.peel += reason · regionNote · source
  • dispose.sprout += reason · cautions · source
  • related[0] → 실제 싹 콘텐츠 (h1 검색어형)

🔧 필터 축은 그대로 (3축 유지)

현재 place 옵션 3개인데 방법은 2개 → 냉장이 빈 필터. 냉장 카드 추가로 3옵션 : 3카드 복구. 새 축 추가 안 함(schema "최대 3축" 준수: place·light·companion).

7 성공 지표

검색 유입"감자 껍질 음식물", "감자 싹 왜", "감자 냉장 보관" → 감자 상세/싹 콘텐츠 랜딩 수.
근거 소비폐기 카드 → 상세 전개(탭) 비율 = 근거를 실제로 읽는가.
AI 인용 (GEO)Perplexity·AI Overviews가 감자 분리배출·싹 답변에 출처로 인용하는지.
필터 이탈 하락place=냉장 클릭 후 이탈률 ↓ (빈 결과 → "왜 안 되는지" 카드).

8 범위 밖 / 미정 — 팀 확인

🟡 팀과 정할 것
  • 싹 콘텐츠 형태 — 별도 라우트(/ingredients/potato/sprout) vs 처리탭 내 확장 블록. 기획은 별도 라우트 가정(SEO 독립 URL 유리).
  • 분리배출 출처 URL — 식약처 정책브리핑(korea.kr, 2021) vs 환경부 "내 손안의 분리배출" 중 정식 출처 고정. 지금은 기관·type만 확정.
  • 냉장 카드 rating — 비추천 카드에 별점을 붙일지. 지금은 장금이 코멘트로 대체, rating 생략 가능.
⛔ 범위 밖
  • 새 화면·컴포넌트 개발 (기존 3탭 목업 재사용).
  • 이미지 에셋 (image:null 유지) · 레시피 UGC 추가.

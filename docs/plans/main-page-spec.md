# 메인 페이지 명세

## 한 줄 정의

장금이 메인 페이지는 사용자가 식자재를 검색하거나 인기 식자재를 선택해 식자재 상세 정보로 빠르게 진입하는 모바일 홈 화면이다.

## 기준 목업

현재 메인 페이지의 기본 화면 틀은 다음 목업을 기준으로 한다.

```text
docs/mockups/main-page-mockup.html
```

이 문서는 위 목업의 화면 구조를 명세화한다. 이후 세부 기능, 데이터 정책, 문구, 인터랙션은 이 문서에 단계적으로 추가한다.

## 목표

- 사용자가 첫 화면에서 장금이의 식자재 탐색 서비스를 이해한다.
- 검색창을 통해 식자재 상세 화면으로 빠르게 이동한다.
- 인기 식자재 카드로 검색 없이도 주요 식자재를 탐색할 수 있게 한다.
- 최근 본 재료를 통해 재방문 흐름을 만든다.
- 장금이 캐릭터와 배너 이미지는 보조 안내 역할로 사용한다.

## 화면 골격

목업은 모바일 앱 셸을 기준으로 한다.

- 상단 앱 헤더(공통 앱 레이아웃 작업물을 사용)
- 검색 영역
- 장금이 소개 배너
- 인기 상승 재료 영역
- 최근 본 재료 영역
- 식자재 관리 안내 배너
- 하단 탭바(공통 앱 레이아웃 작업물을 사용)

데스크톱에서는 모바일 앱 화면을 가운데 배치한 프리뷰 형태로 보여준다. 실제 서비스 구현 시에도 모바일 우선 구조를 기준으로 한다.

## 섹션 명세

### 상단 앱 헤더

- 상단 앱 헤더는 기존 공통 앱 레이아웃 작업물을 사용한다.
- 이번 메인 페이지 작업 범위에서는 상단 앱 헤더 구조와 스타일을 수정하지 않는다.
- 좌측에는 장금이 로고를 노출한다.
- 우측에는 로그인 진입 아이콘을 노출한다.
- 헤더는 화면 상단에 고정된 앱바 역할을 한다.
- 로고 클릭 시 메인 페이지로 이동한다.
- 로그인 아이콘 클릭 시 로그인 화면 또는 로그인 플로우로 이동한다.

### 검색 영역

- 메인 콘텐츠 최상단에 검색창을 배치한다.
- 검색창 왼쪽에는 돋보기 아이콘을 배치한다.
- 돋보기 아이콘 크기는 너비/높이 약 20px을 기준으로 한다.
- 돋보기 아이콘과 input 사이 간격은 약 12px을 기준으로 한다.
- 검색창 좌우 padding은 약 16px을 기준으로 한다.
- Tailwind CSS 사용 시 위 수치와 가까운 유틸리티 값을 사용한다.
- 플레이스홀더는 사용자가 바로 입력할 수 있도록 예시 재료명을 포함한다.
- 예시 문구는 `재료를 검색해보세요 (예: 수박, 양파)`를 기준으로 한다.
- 검색 대상은 식자재 이름과 별칭이다.
- 돋보기 아이콘 클릭 시 현재 입력값 또는 선택된 검색 결과에 해당하는 식자재 상세 화면으로 이동한다.
- 검색 결과 선택 시 해당 식자재 상세 화면으로 이동한다.

### 장금이 소개 배너

- 검색창 아래에 장금이 소개 배너를 배치한다.
- 배너는 장금이 캐릭터 이미지를 배경 또는 보조 이미지로 사용한다.
- 카피는 장금이가 식자재 관리를 도와준다는 인상을 전달한다.
- CTA 문구는 `재료 찾기`를 기준으로 한다.
- `재료 찾기` CTA만 클릭 가능한 링크로 제공한다.
- 배너 전체는 클릭 영역으로 사용하지 않는다.
- `재료 찾기` CTA는 검색 페이지로 이동하는 진입 버튼 역할을 한다.
- 목업 기준 문구는 다음과 같다.

```text
장금이와
신선한 하루 시작!

재료 고르기부터 보관까지 필요한 정보를 쉽게 알려드릴게요.
```

### 인기 상승 재료 영역

- 사용자가 바로 눌러볼 수 있는 식자재 카드 그리드를 제공한다.
- 목업 기준 섹션명은 `인기 상승 재료`다.
- `전체보기` 오른쪽에는 몸통 없는 오른쪽 화살표 아이콘을 넣는다.
- `전체보기` 클릭 시 검색 페이지로 이동한다.
- 인기 상승 재료는 랭킹이 높은 순서대로 12개를 노출한다.
- 한 화면에는 4개 카드를 노출한다.
- 12개 식자재를 4개씩 나누어 총 3페이지 페이지네이션으로 보여준다.
- 각 카드는 식자재 이미지와 식자재명을 포함한다.
- 카드 클릭 시 해당 식자재 상세 화면으로 이동한다.
- 3번째 페이지에서 오른쪽 next 버튼을 누르면 랭킹 1위 식자재가 있는 첫 번째 페이지로 돌아간다.
- 첫 번째 페이지에서 왼쪽 prev 버튼을 누르면 세 번째 페이지로 이동한다.
- 페이지네이션은 처음과 끝이 이어지는 무한 루프 방식으로 동작한다.

목업에 사용된 노출 후보는 임시 예시 데이터다. 실제 인기 상승 재료 12개는 랭킹 산정 기준이 정해진 뒤 데이터 기반으로 노출한다.

목업에 사용된 임시 예시 재료는 다음과 같다.

- 감자
- 양파
- 오이
- 토마토
- 사과
- 양배추
- 당근
- 바나나
- 대파
- 두부
- 달걀
- 버섯

### 최근 본 재료 영역

- 사용자가 최근 방문한 식자재를 보여준다.
- 목업 파일을 만들 때는 최근 본 재료가 추가된 상태로 만들지, 아무것도 추가되지 않은 빈 상태로 만들지 먼저 선택한다.
- 최근 본 재료가 없으면 빈 상태를 보여준다.
- 빈 상태에서는 `아직 본 재료가 없어요` 문구 위에 시계 아이콘을 넣는다.
- 목업 기준 빈 상태 문구는 `아직 본 재료가 없어요`다.
- 최근 본 재료가 생기면 빈 상태 대신 최근 본 재료 목록을 보여준다.
- 최근 본 재료 목록 전체는 점선 프레임 안에 담아 영역을 구분한다.
- 최근 본 재료는 최대 5개까지 노출한다.
- 각 최근 본 재료 항목은 식자재 이미지와 식자재명을 포함한다.
- 각 항목은 동그란 형태의 카드 또는 원형 프레임 안에 배치한다.
- 원형 프레임을 클릭하면 해당 식자재 상세 페이지로 이동한다.
- 최근 본 재료는 왼쪽부터 오른쪽으로 채워진다.
- 5개가 모두 채워진 상태에서 새로운 식자재 상세 화면에 진입하면 가장 왼쪽의 오래된 식자재가 사라진다.
- 새로 본 식자재는 가장 오른쪽 5번째 위치에 추가된다.
- 새로운 식자재가 추가될 때마다 기존 항목은 한 칸씩 왼쪽으로 밀리는 방식으로 동작한다.

### 식자재 관리 안내 배너

- 화면 하단 쪽에 식자재 관리의 서비스 가치를 다시 안내하는 배너를 배치한다.
- 배너 이미지는 신선한 식자재 이미지를 사용한다.
- 구매 이후 보관까지 연결되는 흐름을 짧게 전달한다.
- 목업 기준 문구는 다음과 같다.

```text
사고 끝이 아니라,
신선함까지 챙기는 장금이

장보기부터 냉장고 보관까지 재료의 좋은 순간을 놓치지 않게 도와드려요.
```

### 하단 탭바

- 하단에는 홈, 검색, 로그인 탭을 둔다.
- 현재 페이지는 홈 탭을 활성 상태로 표시한다.
- 검색 탭은 재료 검색 또는 탐색 화면으로 이동한다.
- 로그인 탭은 로그인 화면 또는 로그인 플로우로 이동한다.

## 주요 동작

- 사용자가 검색어를 입력하면 식자재 이름 또는 별칭을 기준으로 검색한다.
- 사용자가 식자재 카드를 선택하면 식자재 상세 화면으로 이동한다.
- 인기 상승 재료 영역의 이전/다음 버튼은 노출 페이지를 순환한다.
- 페이지 도트 선택 시 해당 식자재 카드 묶음을 보여준다.
- 최근 본 재료가 없으면 빈 상태를 유지한다.
- 메인 페이지는 로그인하지 않은 사용자도 탐색할 수 있어야 한다.

## 데이터 연결

식자재 목록의 기본 데이터는 `.codex/skills/content/list.json`의 구조를 따른다.

- `id`: 상세 화면 이동에 사용하는 식자재 slug
- `name`: 카드와 검색 결과에 노출할 이름
- `aliases`: 검색 매칭에 사용할 별칭
- `category`: 분류 표시 또는 필터 확장에 사용
- `emoji`: 이미지가 없을 때 임시 표시로 사용할 수 있는 값
- `summary`: 카드 또는 검색 결과 설명에 사용할 수 있는 짧은 설명
- `seasonMonths`: 제철 표시 또는 추천 기준에 사용할 수 있는 값
- `status`: 공개, 작성 중, 예정 상태 판단
- `hasContent`: 상세 콘텐츠 존재 여부 판단
- `order`: 기본 정렬

목업의 식자재 목록은 하드코딩되어 있으나 실제 구현에서는 데이터 기반 렌더링을 목표로 한다.

## 이미지 자산

메인 페이지에서 사용하는 주요 이미지 자산은 다음 성격을 가진다.

- 로고: `assets/images/logo.webp`
- 장금이 소개 배너: `assets/images/main/janggeumi-banner-character.webp`
- 인기 식자재 카드 이미지: `ingredient-sprite.png`
- 식자재 관리 안내 배너: `assets/images/main/fresh-ingredients-banner.webp`

원본 PNG가 있는 경우 원본은 보존하고, 실제 사용 이미지는 WebP를 기준으로 한다.

이미지 생성 기준은 `.codex/skills/images` 하위 문서를 따른다.

- 식자재 이미지는 `.codex/skills/images/ingredient-image.md`를 따른다.
- 장금이 캐릭터 이미지는 `.codex/skills/images/janggeumi-character.md`를 따른다.
- PNG to WebP 변환은 `.codex/skills/images/png-to-webp.md`를 따른다.

## 라우팅

- 메인 페이지: `/`
- 식자재 상세 화면: `/ingredients/[slug]`
- 재료 페이지: 추후 확정
- 검색/탐색 화면: 추후 확정
- 로그인 화면 또는 플로우: 추후 확정

상세 화면의 구매, 보관, 처리 탭 라우팅은 기존 상세 화면 설계와 맞춘다.

## SEO/GEO 방향

- 메인 페이지는 서비스명과 식자재 탐색 의도를 h1 또는 동등한 주요 제목으로 담아야 한다.
- 검색 가능한 식자재명은 실제 텍스트로 렌더링한다.
- 인기 식자재 카드의 식자재명은 이미지에만 의존하지 않는다.
- 출처 기반 신뢰 정보는 상세 화면에서 다루고, 메인 페이지는 진입과 탐색에 집중한다.

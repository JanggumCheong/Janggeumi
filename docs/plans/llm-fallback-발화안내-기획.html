없는 재료, 장금이가 즉석 발화로 안내

정식 데이터가 없는 재료(없는 상세·검색 무결과)에서 POST /api/ai/chat으로 장금이가 구매·보관·손질을 임시 안내한다. 빈 화면 대신 발화로 공백을 메운다.

문제 (Why)

정식 콘텐츠는 재료 6개뿐(딸기·사과·샤인머스캣·수박·아보카도·천도복숭아). 그 외(당근·양파·고구마…)는 진입 시 404·검색 무결과 → "끝까지 안 버리기" 가치에 어긋난다. 데이터 없다고 빈손으로 돌려보내면 이탈.

대상 & 시나리오

  • 당근 검색 → 매칭 없음 → 장금이 발화
  • 없는 재료 진입 /ingredients/carrot → 404 대신 발화 카드
  • 이상 입력 "asdf" → API가 되묻기("어떤 재료가 궁금하신가요?")

핵심 가치 매핑

끝까지(안 버리기) — 빈손 방지, 발화로라도 안내 빠른 확신 — 구매/보관/손질 구조로 즉시 믿음(집단지성) — 약함: LLM 미검증 → "임시/AI" 명시로 정식과 구분

기획 명세 (spec)

APIPOST /api/ai/chat  {message} → {answer}
message재료 진입=재료명 / 검색=사용자 입력 원문
answer 파싱인트로 → 구매·보관·손질 섹션(- 리스트) → 마무리. 헤더 없으면(되묻기) 통째 1카드
섹션 색구매=초록 / 보관=청록 / 손질=주황 (정식 3탭 액센트 재사용)
AI 표기"임시 안내 · AI" 배지 필수 — 정식(검증) 콘텐츠와 구분
정식 데이터zod 스키마와 무관 — answer 텍스트만 사용
추천 링크정식 재료(수박·딸기 등)로 유도

⚠️ 검색 무분별 호출 방지 (핵심 제약)

LLM은 비용·지연이 있어 타이핑마다 호출 금지. 다층 방어:

  • 명시적 트리거만 — 자동 호출 X, 엔터/검색 버튼 등 확정 액션에만
  • 무결과일 때만 — 매칭 재료 있으면 호출 안 함
  • 디바운스 — 연타 방지 (~500ms)
  • 캐싱 — 같은 쿼리 재호출 방지(쿼리키 캐시), 동일 재료 재진입 시 재호출 X
  • 길이·형식 가드 — 1글자·비정상 입력은 호출 전 컷

트리거

없는 재료 진입→데이터 없음→ai/chat 1회→발화 카드
검색 확정→무결과→ai/chat→발화 카드

로딩 · 에러 (API 스펙: 200 / 422)

  • 로딩: "👩‍🍳 장금이가 생각 중"(LLM 지연 수 초)
  • 422(message 누락·빈값): 호출 전 프론트가 컷
  • 되묻기 답변(200): "어떤 재료가 궁금하신가요?" 그대로 노출(에러 아님)
  • 네트워크·타임아웃·5xx: "지금은 답을 못 드리겠어요 🥲" + 다시 시도
  • 길이 가드: 상한(예 100자) 넘으면 호출 전 컷 (API엔 제한 없음)
  • 강등: 검색發 실패 시 "재료가 없어요"로 우아하게 되돌림

성공 지표

  • 없는 재료·무결과 진입 후 이탈률 감소 (빈 화면 대비)
  • 발화 카드 → 정식 재료 추천 클릭률
  • LLM 호출 수/검색 세션 (무분별 호출 억제 확인 — 낮을수록 좋음)
  • (후속) 발화 상위 재료 = 정식 콘텐츠 우선순위 신호

범위 밖 / 미정 (팀 논의)

  • 연속 대화(채팅 UI) — 이번엔 단발 발화만
  • AI 답변 품질 검수·부적절 답변 처리 방침
  • 정식 콘텐츠 전환 트리거(발화 자주 뜨는 재료 정식화 기준)
  • LLM 호출 상한·레이트리밋 정책 (백엔드 협의)

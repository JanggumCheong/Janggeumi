구매 탭 — 기능 요구사항

#요구사항이유
R1확률 게이지 상단 sticky. 기준 체크 시 확률 실시간 상승, 스크롤해도 고정.토스식 — 핵심 숫자 위 고정 + 즉시 피드백.
R2확률 색 = 상태 기반. 재료 무관 구매색. 0%=중립 회색 → 채울수록 초록(≤33% 주의황).재료(수박·양파…) 바뀌어도 일관. 상태를 색으로.
R3기준 = 2열 원본식. 카드 테두리 없이 구분선+번호, 이미지 우측 뒤 레이어(텍스트가 앞). 행 높이 통일.가벼운 편집 디자인. 텍스트 안 가려지고 정돈됨.
R4기준 선택 = 은은한 배경. 동그라미·박스 없이 카드 토글. 클릭 시 확률 연동.장식 최소. 판정 도구에 폼 체크박스 부적합.
R5Best/Worst 재료 무관 카피. "이런 걸 고르세요/이건 피하세요". 이미지 배경 무채색(사진 선명).재료명 하드코딩 금지. 색은 라벨에만.
R6출처: 무채색 배경 + 행 전체 토글. 요약 항상 노출, 상세만 접기. 유니크 출처 집계, 등급(공식/영상) 구분.E-E-A-T 핵심. 농진청 하드코딩 금지. 확률과 위계 분리.
R7SEO/GEO. h1=검색어, 화면=장금이 몰입 카피. 접힌 출처도 SSR로 DOM에 존재 + JSON-LD(citation).접기는 CSS만 — 크롤러·AI가 읽음. GEO 강화.
R8확률 계산. 현재 6개 균등. 가중치 도입 미정.기준별 중요도 차이 → 논의 필요.

보관 탭 — 기능 요구사항

#요구사항이유
K1신뢰 = 장금이 코멘트. 사용자 후기 대신(로그인·UGC 후속). 카드 하단 전체 폭 한 줄.합의형 근거 필요하나 로그인 전. 검증된 큐레이션이 서비스 가치.
K2방법 비교 우선. 썸네일 크게(72px)로 사진 인지, 순위는 썸네일 좌상단 오버레이. 코멘트는 보조.보관 핵심 = 빠른 방법 비교·선택. 코멘트가 시선 뺏으면 안 됨.
K3태그 무채색. 기간·장소 태그 회색(영역색으로 튀지 않음). 별점은 목업 임시(0 표시 무관).메타 정보는 조용히. 별점은 후속 UGC.
K4필터 = 데이터 3축(form·place·period) 드롭다운. storage.filters와 1:1. '선호도' 등 데이터 없는 필터 금지. 각 방법의 tags로 필터링.필터·콘텐츠 정합성. 재료마다 축이 다름(동적).
K5필터 개수 정책. 5개까지 2줄, 6개+ 혼합/가로스크롤/바텀시트. (수박=3축이라 한 줄)가로 스크롤은 뒤 필터가 숨음. 조합하려면 다 보여야.
K6히어로 = 장금이 화자. 구매 탭과 톤 통일("~보관하는 것이옵니다"). h1은 SEO 검색어.브랜드 몰입 일관 + SEO 병행.

처리 탭 — 기능 요구사항

#요구사항이유
H1활용 / 폐기 2영역. 상단 세그먼트로 갈림. 활용=레시피 UGC, 폐기=분리배출 안내(handling.dispose 신설)."남은 재료 활용"과 "다 쓰고 버리기"는 다른 고민.
H2활용 = 정렬만, 카테고리 필터 없음. "인기순" 등 정렬. 전체/음료/디저트… 칩 제거.확산형은 피드 탐색이 자연스러움. 필터가 탐색 방해.
H3작성 CTA 하단 고정. "나만의 활용법 공유하기". UGC 유도.확산형 = 참여로 콘텐츠가 늘어남.
H4히어로 = 장금이 화자. 구매·보관과 톤 통일.세 탭 브랜드 일관.
